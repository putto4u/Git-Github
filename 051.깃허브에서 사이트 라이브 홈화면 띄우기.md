### 깃허브에서 사이트 라이브 홈화면 띄우기

깃허브의 마크다운(`.md`) 파일은 기본적으로 정적인 텍스트 문서이기 때문에, 웹사이트의 실시간 화면을 라이브 비디오처럼 그대로 스트리밍하여 보여주는 직접적인 기능은 제공하지 않습니다.

하지만 **자동화 도구**를 활용하여 사이트의 최신 홈 화면을 주기적으로 캡처하고, 이를 이미지 링크로 연결하면 사용자가 직접 사진을 찍지 않아도 항상 최신 상태의 화면이 README에 나타나도록 구현할 수 있습니다. 가장 많이 사용되는 효율적인 방법 두 가지를 정리해 드립니다.

---

## 깃허브 마크다운 라이브 스크린샷 구현 방법

### **방법 1. GitHub Actions와 웹샷 서비스 활용 (완전 자동화)**

특정 사이트의 스크린샷을 API 형태로 제공하는 서비스를 깃허브 액션과 조합하는 방식입니다.

1. **동작 원리:** 깃허브 액션이 매일 혹은 매시간마다 특정 사이트를 방문하여 스크린샷을 찍고, 그 이미지 파일을 내 저장소에 업데이트합니다.
2. **구현 단계:**
* `.github/workflows/screenshot.yml` 파일을 생성합니다.
* `puppeteer`나 전용 스크린샷 액션(예: `screenshot-action`)을 사용하여 대상 URL의 화면을 캡처하도록 설정합니다.
* 캡처된 파일을 `git commit` 하여 저장소의 특정 경로(예: `images/latest_home.png`)에 저장합니다.


3. **마크다운 적용:**
```markdown
![최신 사이트 홈 화면](./images/latest_home.png)

```



* **장점:** 내 저장소에 파일이 남으므로 속도가 빠르고 안정적입니다. 사이트 디자인이 바뀌면 자동으로 이미지가 교체됩니다.

---

### **방법 2. 실시간 스크린샷 생성 API 링크 사용 (가장 간편)**

이미지를 직접 저장하지 않고, 호출할 때마다 해당 사이트의 현재 모습을 렌더링해서 보여주는 외부 서비스를 이미지 주소로 사용하는 방법입니다.

* **사용 예시 (Thum.io 또는 유사 서비스):**
```markdown
![실시간 홈 화면](https://image.thum.io/get/https://www.google.com/)

```


* **장점:** 복잡한 설정 없이 URL 한 줄로 즉시 구현 가능합니다. 사용자가 README를 볼 때마다(혹은 서비스의 캐시 주기마다) 최신 화면을 가져옵니다.
* **단점:** 외부 서비스의 로딩 속도에 의존하며, 무료 버전의 경우 워터마크가 있거나 갱신 주기가 길 수 있습니다.

---

### **정리 및 [비용/보안 관련 참고]**

| 구분 | GitHub Actions (추천) | 외부 API 링크 |
| --- | --- | --- |
| **최신성** | 설정한 스케줄 주기마다 갱신 | 호출 시 실시간 갱신 (캐시 적용) |
| **난이도** | 중간 (스크립트 작성 필요) | 매우 쉬움 (URL 복사) |
| **안정성** | 높음 (내 저장소 데이터 사용) | 낮음 (외부 서비스 장애 시 미출력) |

* **[유료 과금 주의]:** * **GitHub Actions:** 공개 저장소(Public)에서는 무료이나, 비공개 저장소(Private)에서 실행 시간이 과도하게 길어지면 매월 제공되는 무료 시간(Minutes)을 초과하여 비용이 발생할 수 있습니다.
* **스크린샷 API:** Thum.io 등 대부분의 서비스는 하루 호출 횟수 제한이 있으며, 대량의 트래픽이 발생하거나 고화질 캡처를 원할 경우 유료 플랜으로 전환해야 합니다.


* **보안:** 로그인 페이지나 개인정보가 포함된 페이지를 실시간 캡처할 경우 민감 정보가 노출될 수 있으니 반드시 사이트 홈 화면 등 공개된 페이지만 대상으로 설정하십시오.
