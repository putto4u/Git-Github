네, 깃허브에서는 다른 저장소의 파일을 단순히 링크로 연결하는 것을 넘어, 내 저장소의 하위 디렉터리처럼 관리하고 목록에 표시할 수 있는 **'Git Submodule(서브모듈)'**이라는 강력한 기능을 제공합니다.

---

## Git Submodule: 다른 저장소를 내 목록에 포함하기

**Git Submodule**은 특정 Git 저장소 안에 다른 Git 저장소를 하위 디렉터리로 포함시키는 방식입니다. 이를 통해 다른 사람의 라이브러리나 공통 자료를 내 저장소의 일부처럼 보여주고 관리할 수 있습니다.

### **1. 서브모듈 추가 방법 (CLI 기준)**

내 저장소 로컬 디렉터리에서 터미널을 열고 다음 명령어를 입력합니다.

```bash
# git submodule add [상대 저장소 URL] [내 저장소 내 폴더명]
git submodule add https://github.com/other-user/external-repo.git external-data

```

* **결과:** 내 저장소 목록에 `external-data`라는 폴더가 생기며, 깃허브 웹 페이지에서는 해당 폴더 아이콘에 화살표가 표시된 형태로 나타납니다. 이를 클릭하면 원본 저장소로 즉시 연결됩니다.

---

### **2. 깃허브 웹에서의 모습**

서브모듈이 정상적으로 추가되면 깃허브 인터페이스에서 다음과 같이 보입니다.

* **폴더 목록:** 일반 폴더와 달리 `folder-name @ commit-hash` 형태로 표시됩니다.
* **연결성:** 해당 폴더를 클릭하면 링크된 원본 저장소의 특정 시점(커밋)으로 이동합니다.

---

### **3. 주요 특징 및 활용 사례**

* **버전 고정:** 원본 저장소가 업데이트되어도 내 저장소는 내가 설정한 특정 시점의 파일들만 가리키도록 고정할 수 있습니다.
* **독립적 관리:** 내 저장소의 소스 코드와 외부 자료를 물리적으로 분리하여 관리할 수 있어 프로젝트가 깔끔해집니다.
* **협업 효율:** 여러 프로젝트에서 공통으로 사용하는 설정 파일이나 라이브러리를 하나의 저장소에 두고 각자의 저장소에서 서브모듈로 링크해 사용합니다.

---

### **4. [참고 및 주의사항]**

1. **복제 시 주의:** 서브모듈이 포함된 저장소를 `git clone` 할 때는 서브모듈 내용까지 한꺼번에 가져오기 위해 `--recursive` 옵션을 붙여야 합니다.
* `git clone --recursive [내 저장소 URL]`


2. **권한 문제:** 만약 링크하려는 저장소가 **비공개(Private)** 저장소라면, 해당 권한이 없는 다른 사용자들은 내 저장소에서 그 서브모듈 내용을 볼 수 없습니다.
3. **[유료 과금 및 할당량]:** 깃허브 서브모듈 사용 자체는 무료입니다. 하지만 서브모듈로 연결된 저장소에 대용량 파일(LFS)이 많고, 이를 빈번하게 내려받는 자동화 빌드(GitHub Actions)를 설정했다면 **LFS 대역폭 할당량**이 소모되어 추가 비용이 발생할 수 있으니 대규모 자료 링크 시 주의하십시오.
